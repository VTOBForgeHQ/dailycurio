# Purpose

The DailyCurio Design System defines the principles, decisions, and reusable patterns that guide the product experience.

It exists to maintain consistency between design and engineering.

---

# Design Philosophy

DailyCurio is built around one idea:

Helping people discover something meaningful every day.

The interface should support curiosity, not compete with it.

# Design Token Ownership

The design system owns:

- Colors
- Typography
- Spacing
- Components
- Interaction patterns

Engineering implements these decisions.

---

# Engineering Relationship

Design answers:

"What should the experience be?"

Engineering answers:

"How should it be built?"

Both collaborate through documented handoffs.

---

# Change Process

Any major design change requires:

1. Document decision.
2. Explain reason.
3. Update affected systems.
4. Communicate engineering impact.
