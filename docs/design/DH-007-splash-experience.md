# Purpose

This document defines the first launch experience of DailyCurio.

The splash experience introduces the product identity before users enter the main application.

---

# Splash Philosophy

The splash screen should feel:

- Calm
- Curious
- Premium
- Simple
- Memorable

The goal is not to entertain users while loading.

The goal is to establish trust and identity.

---

# Experience Goal

When users open DailyCurio, they should immediately understand:

"This is an app built around discovering something meaningful."

---

# Visual Structure

The splash screen contains:

## 1. Brand Mark

Purpose:

Introduce DailyCurio identity.

Requirements:

- Centered.
- Clear.
- Simple.
- Recognizable.

---

## 2. Application Name

Text:

DailyCurio

Purpose:

Build brand recognition.

Typography:

Use approved typography system.

---

## 3. Supporting Message

Purpose:

Communicate the product philosophy.

Initial direction:

"Discover something new every day."

---

# Layout Principles

The splash screen should prioritize:

1. Brand recognition.
2. Calm visual balance.
3. Clear hierarchy.

Avoid:

- Excessive text.
- Multiple actions.
- Visual noise.

---

# Color Usage

Background:

Use approved DailyCurio background color.

Brand elements:

Use approved primary brand color.

No additional colors should be introduced.

---

# Motion Direction

The splash animation should be subtle.

Preferred behavior:

1. Brand mark appears naturally.
2. Application name becomes visible.
3. Transition into the application.

Avoid:

- Fast movement.
- Complex animations.
- Long delays.

---

# Timing Principles

The splash experience should feel intentional.

It should not delay the user unnecessarily.

Loading time should belong to the system.

Animation time should belong to the experience.

---

# Accessibility

The splash experience must remain understandable without animation.

Motion should enhance—not become required.

---

# Engineering Handoff Preview

Engineering implementation will require:

- Splash screen widget.
- Brand assets.
- Typography tokens.
- Color tokens.
- Motion configuration.

Implementation details belong to Engineering.
